# PARALLEL ONLINE ORDER PROCESSING SYSTEM

## INTRODUCTION
This project focuses on demonstrating the concept of parallel programming using an online order processing system. In real-world applications, systems often need to handle multiple tasks simultaneously to improve efficiency. Therefore, this project compares sequential and parallel processing methods to analyze their performance in handling multiple orders.

## OBJECTIVE
- To implement sequential processing in handling orders
- To implement parallel processing using threading
- To compare execution time between sequential and parallel processing
- To understand how parallel programming improves system performance

## METHODOLOGY
The system simulates a number of customer orders. Each order goes through three main process :
- Payment
- Packaging
- Delivery

- Sequential Processing : Each order is process one by one. The next order will only start after previous one is completed.
- Parallel Processing : Multiple orders are process simultaneously using Python threading, allowing faster execution.
